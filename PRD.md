## 1. Обзор продукта

### 1.1 Проблема

Планирование мультисегментного путешествия через российские тревел-агрегаторы (Aviasales, Ostrovok) требует ручного сведения данных из 5-10 вкладок браузера: маршруты, цены, время, багаж, политики отмены, рейтинги отелей. Пользователь тратит часы на компиляцию итогового плана и часто упускает логистические нестыковки (прилёт раньше check-in, багаж на одного при двух пассажирах, отсутствие ночлега между сегментами).

### 1.2 Решение

Claude Code скилл, который:
1. Принимает ссылки на перелёты (Aviasales) и отели (Ostrovok) в свободной форме
2. Автоматически открывает SPA-сайты в Chrome, извлекает структурированные данные
3. Анализирует логистику и находит проблемы
4. Генерирует self-contained HTML-файл с полным маршрутом, экспортом в XLSX и PDF

### 1.3 Целевая аудитория

- Русскоязычные путешественники, самостоятельно планирующие поездки
- Пользователи Claude Code с установленным расширением Claude in Chrome
- Фокус: мультисегментные поездки (2+ города, 2+ перелёта)

### 1.4 Ключевые метрики успеха

| Метрика | Цель |
|---------|------|
| Время от ссылок до готового HTML | < 5 минут на 4 перелёта + 3 отеля |
| Точность извлечения цен | 100% (цены — критические данные) |
| Обнаружение логистических проблем | 100% для 5 основных типов проверок |
| Пользовательские правки после генерации | < 2 ручных корректировок |

## 2. Функциональные требования

### 2.1 Ввод данных

#### FR-1: Автоматический триггер
**Приоритет:** P0
Скилл активируется автоматически при:
- Вставке ссылок `avs.io/*`, `aviasales.ru/*`, `corp.ostrovok.ru/*`
- Даже без текста — одних ссылок достаточно
- Команды на русском: "обнови таблицу", "добавь рейс", "поменяй отель", "проверь цены"
- Русские тревел-термины: отпуск, поездка, перелёт, отель, бронирование, маршрут

#### FR-2: Парсинг входных данных
**Приоритет:** P0
- Извлечение коротких ссылок Aviasales (`avs.io/*`) с получением redirect URL
- Парсинг URL-параметров Ostrovok (даты, кол-во гостей, регион)
- Извлечение дат, направлений, количества гостей из свободного текста
- Поддержка скриншотов — визуальное чтение данных с экранов Aviasales/Ostrovok

### 2.2 Извлечение данных о перелётах (Aviasales)

#### FR-3: Двухфазное извлечение
**Приоритет:** P0

**Фаза A — Быстрые данные из redirect URL:**
- `WebFetch` на `avs.io` ссылку получает 301 → полный `aviasales.ru` URL
- Из URL извлекаются: `expected_price`, `expected_price_currency`, код маршрута, код авиакомпании

**Фаза B — Полные данные из браузера:**
- Открытие ссылки через `mcp__claude-in-chrome__navigate`, ожидание 2 сек
- Автоматическое открытие попапа с деталями билета
- Запуск JS-экстрактора для извлечения данных из DOM

#### FR-4: Извлекаемые данные о перелёте
**Приоритет:** P0

| Поле | Описание |
|------|----------|
| Маршрут | Коды аэропортов + названия городов |
| Дата/Время | Вылет, прилёт, длительность |
| Авиакомпания | Название + тип (регулярный/чартер) |
| Пересадки | Кол-во, аэропорты, время стыковки |
| Багаж | Ручная кладь (вес x кол-во), зарег. багаж (вес x кол-во), включён/доплата |
| Возврат/обмен | Политика: платный, бесплатный, невозможен |
| Цена | Общая (на всех пассажиров) + список продавцов с ценами |

#### FR-4a: Соседние гейтвеи и прямые чартеры
**Приоритет:** P1
- Перед фиксацией маршрута предлагать 1–2 альтернативных гейтвея (соседний город/аэропорт) и прямые чартеры из РФ (Azur / Nordwind / Red Wings) — самый дешёвый/быстрый вход не всегда через ближайший аэропорт (напр. прямого MOW→JRO нет, а прямой MOW→ZNZ есть + дешёвый хоп до Аруши).
- По согласию юзера прогонять доп-поиски и сравнивать; смотреть «цены на соседние даты» (±1–3 дня). Не перемаршрутизировать молча.

### 2.3 Извлечение данных об отелях (Ostrovok)

#### FR-5: Браузерное извлечение
**Приоритет:** P0
- Навигация на URL отеля через Chrome
- Ожидание гидратации SPA (title меняется с "Загрузка отеля..." на название отеля)
- Запуск JS-экстракторов для цен, типов номеров, рейтингов

#### FR-6: Извлекаемые данные об отеле
**Приоритет:** P0

| Поле | Описание |
|------|----------|
| Название | + кол-во звёзд |
| Адрес | Район, город |
| Check-in/Check-out | Время заезда и выезда |
| Типы номеров | Все доступные: название, площадь, тип кровати |
| Питание | Тип (завтрак, полупансион, AI) |
| Отмена | Политика + дедлайн бесплатной отмены |
| Цена | За номер (на всех гостей) |
| Рейтинг Ostrovok | Оценка/10 + кол-во отзывов |
| TripAdvisor | Прямая ссылка, рейтинг/5, кол-во отзывов |

#### FR-6a: Выбор портала Ostrovok (corp vs публичный)
**Приоритет:** P1
- Два входа: `corp.ostrovok.ru` (портал «Командировки» — нужен агентский логин; нетто-цены в ₽, оплата рос-картой, иногда шире инвентарь, часто дешевле официального сайта отеля) и публичный `ostrovok.ru` (гостевые цены, без логина). Скилл шарится — в начале работы с отелями спросить один раз, какой использовать; default — публичный.
- Не угадывать slug региона (отель может лежать под неожиданным, напр. Riu → `kendwa`) — вбивать название в поиск/автокомплит и брать подсказку (правильный регион + `mid`).

### 2.4 Трансферы

#### FR-7: Оценка трансферов
**Приоритет:** P1
- Автоматическая вставка строк трансфера между аэропортом и отелем
- Встроенная база времени для турецких маршрутов:
  - Istanbul IST -> центр: 1-1.5ч
  - Nevsehir NAV -> Goreme: 20-30 мин
  - Dalaman DLM -> Fethiye: 1-1.5ч
  - Dalaman DLM -> Marmaris: 1.5ч
  - Antalya AYT -> Kemer: 1ч
  - Antalya AYT -> Side: 1.5ч
  - Antalya AYT -> Alanya: 2.5ч

### 2.5 Логистический анализ

#### FR-8: Проверки
**Приоритет:** P0

| Проверка | Описание |
|----------|----------|
| Accommodation gaps | Ночи без забронированного отеля |
| Check-in timing | Прилёт раньше check-in с учётом трансфера |
| Check-out timing | Недостаточно времени: checkout + трансфер + 2ч запас до вылета |
| Baggage mismatch | Мест багажа меньше, чем пассажиров |
| Non-refundable deadlines | Дедлайны невозвратных бронирований |

### 2.6 Генерация HTML

#### FR-9: Структура файла
**Приоритет:** P0
- Путь: `~/Desktop/trip_[destination]_[dates].html`
- Self-contained: inline CSS, без внешних стилей
- SheetJS загружается с CDN для XLSX-экспорта

#### FR-10: Таблица маршрута
**Приоритет:** P0
- Цветовое кодирование: синий (#0071e3) — перелёты, зелёный (#34c759) — отели, оранжевый (#ff9500) — трансферы
- 9 колонок: #, Тип, Дата, Описание, Время/Заезд-Выезд, Детали, Рейтинг, Цена, Ссылки
- TripAdvisor рейтинг с зелёным бейджем и иконкой
- Ссылки на Ostrovok и TripAdvisor для каждого отеля

#### FR-11: Summary card
**Приоритет:** P0
- Общая стоимость
- Подитог перелётов + подитог отелей
- Длительность поездки + количество ночей

#### FR-11a: Budget card (смета, опц.)
**Приоритет:** P1
- Построчная смета (`budget`: `items[]` + один выделенный `total`) для категорий вне flights/hotels totals — тур/сафари-оператор, трансферы, виза, страховка, чаевые. Рендерится карточкой после summary, попадает в XLSX, из price-free PDF исключён. Диапазон в `total.value` для оценочных цифр. Опционально: нет `budget` → вывод не меняется.

#### FR-12: Notes card
**Приоритет:** P0
- Предупреждения о логистике (красным)
- Подтверждения (зелёным, "стыкуется")
- Примечания о багаже
- Дедлайны отмены бронирований
- Дисклеймер о ценах

### 2.7 Экспорт

#### FR-13: XLSX экспорт
**Приоритет:** P1
- 18 колонок: #, Тип, Дата, Описание, Маршрут, Время, В пути, Оператор, Номер/Класс, Питание, Условия отмены, Багаж, Цена, TripAdvisor, Отзывов, Ostrovok, Ссылка бронирования, Ссылка TripAdvisor
- Итоговые строки: перелёты, отели, общий итог
- Автоширина колонок
- Файл: `[Destination]_Trip_[Dates].xlsx`

#### FR-14: PDF экспорт
**Приоритет:** P1
- Mobile-optimized layout (max-width: 420px)
- Карточная раскладка, сгруппированная по дням
- Без цен — только логистика, ссылки, рейтинги
- `break-inside: avoid` для корректной печати
- Автозапуск `window.print()` через 400мс

### 2.8 Память о поездках

#### FR-15: Персистентный реестр поездок
**Приоритет:** P1
- Скилл помнит каждую сгенерированную поездку **между сессиями и агентами** — стор вне каталога плагина: `~/.trip-planner/` (override `$TRIP_PLANNER_HOME`), переживает `claude plugin update`.
- **Recall** (перед Step 0): любой агент читает реестр, отвечает на вопросы об истории, детектирует дубликаты, де-приоритизирует `archived`.
- **Record** (Step 9): после генерации HTML поездка пишется в реестр (idempotent upsert по `id`; авто-захват полей и структурного `trip-data` из HTML).
- `trips.json` — canonical (single-writer `trip_registry.py`, atomic + `flock`); `trips.md` — авто-генерируемое человекочитаемое зеркало.
- Команды: `record`, `list [--status]`, `get`, `remove`, `render` (re-emit HTML из кэша без рескрейпинга), `status`, `merge`. Stdlib-only.
- Жизненный цикл: `planned` → `booked` → `archived`.

#### FR-16: Структурный single-source-of-truth в HTML
**Приоритет:** P2
- Данные поездки живут в одном `<script id="trip-data" type="application/json">` блоке; таблица, summary, notes, XLSX и PDF рендерятся из него (устраняет рассинхрон 4 секций).
- Редактирование одной даты/отеля не требует рескрейпинга: правка JSON-блока (или `render` из реестра) обновляет все представления.

## 3. Нефункциональные требования

### NFR-1: Производительность
- Извлечение данных одного перелёта: < 30 сек
- Извлечение данных одного отеля: < 30 сек
- Генерация HTML: < 5 сек
- Общее время (4 перелёта + 3 отеля): < 5 мин

### NFR-2: Надёжность
- Fallback при сбое JS-экстрактора: скриншот страницы + визуальное чтение
- Ожидание гидратации SPA: проверка title/контента перед извлечением
- Устойчивость к изменениям DOM: экстракторы ищут по контенту (тексту), а не по CSS-селекторам

### NFR-3: Совместимость
- macOS + Chrome + расширение Claude in Chrome
- HTML-вывод: все современные браузеры (Chrome, Safari, Firefox, Edge)
- XLSX: совместимость с Excel, Google Sheets, Numbers
- PDF: стандартный `window.print()`, работает в любом браузере

### NFR-4: Локализация
- Интерфейс: русский язык
- Поддержка рублёвых цен (₽)
- Даты в формате DD.MM (российский стандарт)
- Дни недели на русском

## 4. Ограничения и зависимости

### 4.1 Внешние зависимости

| Зависимость | Тип | Критичность |
|-------------|-----|-------------|
| Claude Code | Runtime | Критическая — без него скилл не работает |
| Claude in Chrome | MCP Extension | Критическая — SPA-сайты не извлекаются без браузера |
| Aviasales DOM | External API | Высокая — изменения DOM могут сломать экстракторы |
| Ostrovok DOM | External API | Высокая — аналогично |
| SheetJS CDN | Library | Средняя — только для XLSX-экспорта |

### 4.2 Известные ограничения

1. **Только Турция** для автоматических трансферов — для других стран нужно вручную добавлять маршруты
2. **Цены — snapshot** — данные актуальны на момент проверки, Aviasales показывает новый поиск по shared-ссылке
3. **Только русский** интерфейс и русскоязычные сайты
4. **Нет persistent storage** — каждый вызов генерирует новый файл, нет базы предыдущих поездок
5. **Чартерные рейсы** имеют отличные правила багажа и возврата, требуют ручной проверки

### 4.3 Риски

| Риск | Вероятность | Влияние | Митигация |
|------|------------|---------|-----------|
| Изменение DOM Aviasales | Высокая (SPA обновляется часто) | JS-экстракторы перестают работать | Fallback на скриншот + визуальное чтение; экстракторы ищут по тексту, не по селекторам |
| Изменение DOM Ostrovok | Средняя | Аналогично | Аналогично |
| SheetJS CDN недоступен | Низкая | XLSX-экспорт не работает | HTML-таблица остаётся доступной, можно скопировать данные вручную |
| Chrome extension disconnected | Средняя | Нельзя извлечь данные | Пользователь получает инструкцию переподключить расширение |

## 5. Архитектура

### 5.1 Тип продукта
**Prompt-based Claude Code skill** — без бэкенда, без базы данных, без серверной инфраструктуры.

### 5.2 Компоненты

```
[Пользователь] → [Claude Code CLI] → [SKILL.md инструкции]
                                            ↓
                                    ┌───────┴────────┐
                                    ↓                ↓
                            [WebFetch]        [Chrome MCP]
                            (redirect URL)    (SPA навигация)
                                    ↓                ↓
                            [Quick price]    [JS Extractors]
                                    ↓                ↓
                                    └───────┬────────┘
                                            ↓
                                    [Logistics Engine]
                                    (проверки стыковок)
                                            ↓
                                    [HTML Generator]
                                    (self-contained файл)
                                            ↓
                                    [~/Desktop/trip_*.html]
                                            ↓
                              ┌─────────────┼─────────────┐
                              ↓             ↓             ↓
                          [Browser]     [XLSX export]  [PDF export]
                                        (SheetJS)      (window.print)
```

### 5.3 Файловая структура

```
trip-planner-skill/
├── SKILL.md              # Ядро скилла: инструкции + JS-экстракторы + workflow
├── example-output.html   # Эталонная реализация HTML-вывода
├── README.md             # Публичная документация
├── PRD.md                # Этот документ
├── DOCS.md               # Техническая документация
├── context-map.md        # Карта контекста для быстрого входа
└── .superset/
    └── config.json       # Конфигурация хуков (пока пустая)
```

## 7. Глоссарий

| Термин | Определение |
|--------|-------------|
| SPA | Single Page Application — сайт, который рендерится на клиенте (Aviasales, Ostrovok) |
| MCP | Model Context Protocol — способ подключения внешних инструментов к Claude |
| Claude in Chrome | Расширение Chrome, дающее Claude Code доступ к DOM браузера |
| WebFetch | Инструмент Claude Code для HTTP-запросов (аналог curl) |
| JS-экстрактор | JavaScript-сниппет, запускаемый в контексте страницы для извлечения данных |
| SheetJS | Библиотека для генерации XLSX-файлов в браузере |
| avs.io | Сервис коротких ссылок Aviasales |
| Гидратация SPA | Момент, когда JavaScript на странице завершает рендер и DOM содержит реальные данные |
